# XO VR Build Plan

This plan keeps future development organized room by room. Each step should be implemented as one focused commit or pull request so the prototype stays easy to review.

## Development Rule

Do not build every room at once. Add one room, object system, hidden mechanic, or content feature per commit. Each commit should leave the app runnable with `npm run dev`.

## Phase 1: Stabilize The Core Room

### Commit 1: Core Room Content Pass

- Refine the current core room around XO Web / network identity.
- Add editable data for the core room objects.
- Keep the heartbeat core as the central symbol.
- Place references to Galata as founding myth, The Locals, Live Music Video, and XO Merch / NPC mechanic.

### Commit 2: Core Room Interaction Pass

- Improve hover and gaze feedback.
- Make social links and core-room project objects easier to edit.
- Add clearer symbolic labels only when users interact.
- Preserve calm, atmospheric pacing.

### Commit 3: XO NPC Placeholder

- Add a simple NPC-like presence or marker tied to XO Merch and hidden-gem behavior.
- Keep it subtle and prototype-level.
- Use placeholder copy and comments for future behavior.
